# ユーザーロール（権限）の仕様書

本システムにおけるユーザー権限の定義と、それぞれの操作可能範囲についてのまとめです。

---

## 1. 本部管理者 (HQ_ADMIN)
全店舗およびシステム全体を管理する最上位の権限です。

- **管理範囲**: 全店舗、全ユーザー。
- **主な可能操作**:
    - **お知らせ**: 全店舗・全ユーザー向けのお知らせ配信、および全投稿の管理（編集・削除）。
    - **ユーザー管理**: 全ユーザーの登録・修正・削除、パスワードリセット、ロールの変更。
    - **画面管理**: ホーム画面に表示されるタイル（アンケート・リンク等）の作成・配置・表示設定.
    - **相談（メール）**: 全店舗に届いた相談内容の確認および対応。
    - **統計**: システム全体の利用状況、お知らせの既読率、アンケートの回答結果などの閲覧。

## 2. 店舗管理者 (STORE_ADMIN)
特定の店舗に紐付いた管理者権限です。（店長・責任者など）

- **管理範囲**: 所属する自店舗、およびその店舗に所属するスタッフ。
- **主な可能操作**:
    - **お知らせ**: 自店舗スタッフに限定したお知らせの配信、および自店舗宛て通知の既読状況確認。
    - **ユーザー管理**: 自店舗スタッフの情報確認（※設定により表示のみ、または一部編集可）。
    - **相談（メール）**: 自店舗に届いた相談への返信・ステータス管理。
    - **統計**: 自店舗に関連するお知らせの反応状況確認。

## 3. 一般ユーザー (GENERAL)
現場スタッフや一般社員が使用する、閲覧とアクションが中心の権限です。

- **管理範囲**: 管理画面 (`/admin`) へのアクセス権はありません。
- **主な可能操作**:
    - **お知らせ**: 配信された情報の閲覧、いいね、コメント投稿、既読付与、しおり作成。
    - **メニュー活用**: ホーム画面のタイルから、アンケートへの回答や外部リンクの利用。
    - **相談（メール）**: 本部または店舗의担当者への相談・報告の送信、および自身の相談履歴の確認。

---

## ロール階層とアクセス制御 (RBAC)
システム内部では以下の階層（ヒエラルキー）で制限をかけています。

**一般 (`GENERAL`) < 店舗管理者 (`STORE_ADMIN`) < 本部管理者 (`HQ_ADMIN`)**

- **URL制限**: `/admin/*` のパスは `STORE_ADMIN` 以上のロールがないとサーバー側で遮断されます。
- **データの分離 (Multi-tenancy)**: 
    - `STORE_ADMIN` が取得するデータには、常に `store_id` によるフィルタリングがかかります。
    - `HQ_ADMIN` の場合は `store_id` のフィルタが無効化され、全件取得となります。
- **機能制限**: タイルの編集（画面管理）などは、現在 `HQ_ADMIN` のみが権限を持つよう実装されています。
